# PostgreSQL to MongoDB Migration Guide

## Overview
Your Purchase Order System has been successfully migrated from PostgreSQL (JPA/Hibernate) to MongoDB. This guide explains the changes made and how to set up and use the MongoDB backend.

## Changes Made

### 1. Dependencies Updated (pom.xml)
- **Removed**: `spring-boot-starter-data-jpa`, PostgreSQL driver
- **Added**: `spring-boot-starter-data-mongodb`, MongoDB Java driver

### 2. Entity Classes Converted to MongoDB Documents
All entities now use MongoDB annotations instead of JPA:
- `@Entity` → `@Document(collection = "collectionName")`
- `@GeneratedValue` → Removed (MongoDB generates ObjectId automatically)
- `@Column` → Removed (MongoDB is schema-less)
- `@Table` → `@Document` with collection name
- `@ManyToOne` → `@DBRef` (for referenced documents)
- `@OneToMany` → `@DBRef` on List (for referenced documents)
- `@PrePersist/@PreUpdate` → `@CreatedDate/@LastModifiedDate` from Spring Data
- `@Id` → `@Id` (String type, auto-generated by MongoDB as ObjectId)

**Document Collections**:
- `purchaseOrders` - Main purchase order documents
- `suppliers` - Supplier references
- `buyers` - Buyer references
- `products` - Product references
- `brands` - Brand references
- `categories` - Category references
- `order_items` - Order line items
- `data_imports` - Import history records

### 3. Repository Changes
- `JpaRepository` → `MongoRepository<Entity, String>`
- Custom `@Query` annotations converted to MongoDB query syntax
- Method signatures updated to use String IDs instead of Long

### 4. Configuration
- **File**: `application.properties`
- **Key Settings**:
  ```properties
  spring.data.mongodb.uri=mongodb://localhost:27017/intosourcing
  spring.data.mongodb.database=intosourcing
  spring.data.mongodb.auto-index-creation=true
  ```

### 5. Controllers Updated
All controllers now accept String IDs instead of Long:
- `getPurchaseOrderById(String id)` instead of `getPurchaseOrderById(Long id)`
- `getSupplierAnalytics(String supplierId)` instead of `getSupplierAnalytics(Long supplierId)`

### 6. DTOs Updated
- `PurchaseOrderDTO`: Changed all IDs from `Long` to `String`
- `OrderItemDTO`: Changed all IDs from `Long` to `String`

## Setup Instructions

### Prerequisites
- MongoDB 4.0+ installed and running
- Java 21
- Maven 3.8+

### Option 1: Local MongoDB Setup (macOS)

#### Using Homebrew
```bash
# Install MongoDB
brew tap mongodb/brew
brew install mongodb-community

# Start MongoDB service
brew services start mongodb-community

# Verify MongoDB is running
mongosh  # MongoDB shell

# Create database and user (optional)
use intosourcing
```

#### Using Docker (Recommended)
```bash
# Pull MongoDB image
docker pull mongo:latest

# Run MongoDB container
docker run -d \
  -p 27017:27017 \
  -e MONGO_INITDB_ROOT_USERNAME=admin \
  -e MONGO_INITDB_ROOT_PASSWORD=admin123 \
  -e MONGO_INITDB_DATABASE=intosourcing \
  --name mongo-intosourcing \
  mongo:latest

# Verify container is running
docker ps | grep mongo-intosourcing
```

#### Update application.properties for Docker MongoDB
```properties
# If using authentication
spring.data.mongodb.uri=mongodb://admin:admin123@localhost:27017/intosourcing
# If without authentication (default)
spring.data.mongodb.uri=mongodb://localhost:27017/intosourcing
```

### Option 2: MongoDB Atlas Cloud (Recommended for Production)

1. Go to [MongoDB Atlas](https://www.mongodb.com/cloud/atlas)
2. Create a free account
3. Create a new cluster
4. Get the connection string
5. Update `application.properties`:
   ```properties
   spring.data.mongodb.uri=mongodb+srv://username:password@cluster.mongodb.net/intosourcing
   ```

### Option 3: Update docker-compose.yml

Replace or add MongoDB service to your existing docker-compose.yml:

```yaml
version: '3.8'

services:
  mongodb:
    image: mongo:latest
    container_name: intosourcing-mongo
    ports:
      - "27017:27017"
    environment:
      MONGO_INITDB_DATABASE: intosourcing
      # Uncomment for production with authentication
      # MONGO_INITDB_ROOT_USERNAME: admin
      # MONGO_INITDB_ROOT_PASSWORD: secure_password
    volumes:
      - mongodb_data:/data/db
      - mongodb_config:/data/configdb
    networks:
      - intosourcing-network

  application:
    build: .
    container_name: intosourcing-app
    ports:
      - "8080:8080"
    environment:
      - SPRING_DATA_MONGODB_URI=mongodb://mongodb:27017/intosourcing
      - SPRING_DATA_MONGODB_DATABASE=intosourcing
    depends_on:
      - mongodb
    networks:
      - intosourcing-network

volumes:
  mongodb_data:
  mongodb_config:

networks:
  intosourcing-network:
    driver: bridge
```

Then run:
```bash
docker-compose up -d
```

## Data Migration from PostgreSQL to MongoDB

### Option 1: Using MongoDB Import Tools

#### Step 1: Export PostgreSQL data as JSON
```bash
# Install mongo-database-tools if not present
# macOS: brew install mongodb-database-tools
# Linux: sudo apt-get install mongodb-database-tools

# Export collections from PostgreSQL
psql -U postgres -d intosourcing -c "COPY (SELECT * FROM suppliers) TO STDOUT WITH CSV HEADER" > suppliers.csv

# Convert CSV to JSON (use a tool like csvtojson)
npm install -g csvtojson
csvtojson suppliers.csv > suppliers.json
```

#### Step 2: Import to MongoDB
```bash
# Import collections to MongoDB
mongoimport --uri "mongodb://localhost:27017/intosourcing" \
  --collection suppliers \
  --file suppliers.json \
  --jsonArray

mongoimport --uri "mongodb://localhost:27017/intosourcing" \
  --collection brands \
  --file brands.json \
  --jsonArray

# ... repeat for other collections
```

## Troubleshooting

### MongoDB Connection Issues
```
Error: com.mongodb.MongoServerSelectionException

Solution:
1. Verify MongoDB is running: mongosh
2. Check connection URI in application.properties
3. Verify firewall allows port 27017
```

### Data Not Persisting
```
Solution:
1. Check MongoDB service status
2. Verify database name in application.properties
3. Check application logs for errors
```

### Slow Queries
```
Solution:
1. Ensure all @Indexed annotations are applied
2. Check MongoDB index creation: db.collection.getIndexes()
3. Use MongoDB Compass or mongosh to analyze query performance
```

## Performance Considerations

1. **@DBRef vs Embedded Documents**:
   - Currently using `@DBRef` for relationships (normalized approach)
   - For better performance on specific queries, consider embedding small documents

2. **Aggregation Pipeline**:
   - Complex analytics queries can use MongoDB aggregation pipelines
   - Currently handled via Java streams; consider migration for high-volume data

3. **Transactions**:
   - MongoDB transactions are supported but may have limitations
   - Multi-document transactions require replica sets
